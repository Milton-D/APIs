¿QUÉ ES UN COMPONENTE EN REACT Y CÓMO SE CREA UNO?

  • Un componente en React es una pieza reutilizable de interfaz de usuario que nos ayuda a estructurar y a hacer modelos de nuestra aplicación. Hay componentes de dos tipos: componentes de clase y componentes funcionales. Los componentes de clase los usamos cuando se necesita un estado interno y para los métodos del ciclo de vida. Los componentes funcionales son más versátiles a la hora de renderizar JSX según las propiedades que reciba. Para la creación de nuestro componente primeramente hacemos la importación de la librería de React en nuestro código, después miramos en que lo vamos a enfocar (de clase o funcional) ya con esto si decidimos clase hacemos la implementación del método render() dentro de la clase que nos devolverá los elementos JSX que tiene mi componente. También se puede definir otros métodos para el ciclo de vida o métodos personalizados dentro de mi clase, esto según el uso que tendrá.

¿CUÁL ES LA DIFERENCIA ENTRE REACTDOM.RENDER() y REACTDOM.HYDRATE()?

  • ReactDOM.render() este esta enfocado en el lado del cliente mientras que ReactDOM.hydrate() es utilizado en lado del servidor para hidratar un html

¿QUÉ ES LA PROP CHILDREN Y CÓMO SE USA EN REACT?

  • En React, la prop children se considera una estructura de datos opaca. Esto significa que no debes confiar en cómo está estructurada. Para transformar, filtrar, o contar children, deberías usar los métodos Children.
    En la practica, la estructura de datos children a menudo se representa como un array internamente. Sin embargo, si solo hay un hijos, entonces React no creará un array extra ya que esto conduciría a una sobrecarga de memoria innecesaria. Siempre y cuando uses los métodos Children en lugar de hacer una introspección directa de los prop children, tú código no se romperá incluso si React cambia la forma en que se implementa realmente la estructura de datos.

¿CÓMO SE PASA INFORMACIÓN ENTRE COMPONENTES EN REACT?

  • Los componentes de React utilizan props para comunicarse entre sí. Cada componente padre puede enviar información a sus componentes hijos mediante el uso de props. Las props pueden parecerte similares a los atributos HTML, pero permiten pasar cualquier valor de JavaScript a través de ellas, como objetos, arrays y funciones.
